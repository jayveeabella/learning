Contextual Bandits

An interactive deep-dive — from LinUCB (Li et al. 2010) to Thompson Sampling and neural extensions

Extends: ← Multi-Armed & Contextual Bandits

In the standard multi-armed bandit, you pick arms and learn which one has the highest average reward. Every round is identical — there's no side information. In the real world, this almost never holds: you know something about the user, the situation, the item.

The contextual bandit adds one thing: before each pull, you observe a context $x_t \in \mathbb{R}^d$ that describes the current situation. The best arm now depends on $x_t$.

Examples: "this user likes sports" → show a sports article. "tumor size is large" → recommend aggressive treatment. "it's Friday evening" → suggest a comedy show.

The formal protocol at each round $t$:

The contextual regret measures loss against the best context-dependent policy:

$$\text{Regret}(T) = \sum_{t=1}^{T} \Bigl(\max_a\, x_t^\top\theta_a^* \;-\; x_t^\top\theta_{A_t}^*\Bigr)$$

Unlike standard MAB regret, the oracle changes every round — it's the arm that's best for this specific context.

⚠️ Not full RL. The context $x_t$ is observed but not affected by your action. There is no state transition — no credit assignment across time. This tractability is what makes contextual bandits viable for real-time production systems.

The Linearity Assumption

Most tractable contextual bandit algorithms assume the expected reward is linear in the context:

$$\mathbb{E}[r_t \mid x_t, a] = x_t^\top \theta_a^* \quad \text{for some unknown } \theta_a^* \in \mathbb{R}^d$$
What this buys you:
  • Closed-form posterior updates (conjugate Gaussian)
  • Efficient $O(d^2)$ per-round computation
  • Provable $O(d\sqrt{T})$ regret bounds
  • The $d$ arms share no parameters — each arm $a$ has its own $\theta_a^*$
What can go wrong:
  • True reward is nonlinear → confidence bounds miscalibrated
  • Model bias doesn't vanish — can cause linear regret
  • Fix: use richer features (polynomial, kernel, neural embeddings)
  • NeuralUCB/NeuralTS handle nonlinearity at higher compute cost

LinUCB: Full Derivation

From ridge regression to the confidence ellipsoid

We have $K$ arms and $T$ rounds. At round $t$, the agent observes context $x_t \in \mathbb{R}^d$ and must choose arm $A_t \in \{1,\ldots,K\}$. The reward is:

$$r_t = x_t^\top \theta_{A_t}^* + \eta_t$$

where $\theta_a^* \in \mathbb{R}^d$ is the unknown true parameter for arm $a$, and $\eta_t$ is $R$-sub-Gaussian noise (zero-mean, e.g. bounded rewards $r \in [0,1]$ with $R=1/2$).

Intuition: Each arm has a "preference vector" $\theta_a^*$ that defines which users/contexts it's good for. A news article about sports has high $\theta_{\text{sports}}$ in the "user likes sports" direction. LinUCB learns these preference vectors online.

Assumptions:

  • Realizability: $\mathbb{E}[r_t \mid x_t, A_t] = x_t^\top \theta_{A_t}^*$ exactly
  • Bounded parameters: $\|\theta_a^*\|_2 \leq S$ for all $a$
  • Bounded contexts: $\|x_t\|_2 \leq L$ for all $t$ (typically normalize to $L=1$)
  • Sub-Gaussian noise: $\mathbb{E}[e^{\lambda\eta_t}|\mathcal{F}_{t-1}] \leq e^{\lambda^2 R^2/2}$ for all $\lambda$

The goal is to minimize the cumulative contextual regret:

$$\text{Regret}(T) = \sum_{t=1}^T \left(\max_a x_t^\top\theta_a^* - x_t^\top\theta_{A_t}^*\right)$$

For a fixed arm $a$, suppose we've collected observations $\{(x_s, r_s)\}_{s \in \mathcal{T}_a}$ where $\mathcal{T}_a$ is the set of rounds where arm $a$ was pulled. We need to estimate $\theta_a^*$.

L2-Regularized Least Squares (Ridge Regression)

Minimize the regularized squared loss:

$$\hat{\theta}_a = \arg\min_{\theta} \sum_{s \in \mathcal{T}_a}(r_s - x_s^\top\theta)^2 + \lambda\|\theta\|_2^2$$

Taking the gradient and setting to zero:

$$\frac{\partial}{\partial\theta}\Bigl[\sum_s(r_s - x_s^\top\theta)^2 + \lambda\|\theta\|^2\Bigr] = -2\sum_s x_s(r_s - x_s^\top\theta) + 2\lambda\theta = 0$$ $$\underbrace{\Bigl(\sum_s x_s x_s^\top + \lambda I\Bigr)}_{A_a}\,\theta = \underbrace{\sum_s r_s x_s}_{b_a}$$
Closed form: Define the key quantities for arm $a$: $$A_a = \lambda I + \sum_{s \in \mathcal{T}_a} x_s x_s^\top \in \mathbb{R}^{d \times d}, \qquad b_a = \sum_{s \in \mathcal{T}_a} r_s x_s \in \mathbb{R}^d$$ $$\boxed{\hat{\theta}_a = A_a^{-1} b_a}$$

Why $\lambda I$? The regularization term serves three purposes:

  • Invertibility: $A_a$ is always positive definite ($A_a \succeq \lambda I$), even with zero observations
  • Bayesian prior: Equivalent to a Gaussian prior $\theta_a \sim \mathcal{N}(0, \lambda^{-1}I)$ — at $\lambda=1$ this assumes unit-norm parameters
  • Calibration: The confidence ellipsoid's radius (see Tab 4) depends on $\lambda$ — it must match the prior strength used

Starting state: Before any pulls, $A_a = \lambda I$ and $b_a = \mathbf{0}$, giving $\hat\theta_a = \mathbf{0}$. This is the maximum-uncertainty state.

In the online setting, we update $A_a$ and $b_a$ one observation at a time. When round $t$ selects arm $A_t = a$ and receives reward $r_t$:

$$A_a \leftarrow A_a + x_t x_t^\top, \qquad b_a \leftarrow b_a + r_t x_t$$

This is a rank-1 update to $A_a$. We then need $A_a^{-1}$ for the UCB score.

Sherman-Morrison Formula for $O(d^2)$ Updates

Recomputing $A_a^{-1}$ from scratch each round costs $O(d^3)$ (matrix inversion). The Sherman-Morrison formula provides a $O(d^2)$ rank-1 inverse update:

$$(A + uv^\top)^{-1} = A^{-1} - \frac{A^{-1}uv^\top A^{-1}}{1 + v^\top A^{-1}u}$$

Applied with $u = v = x_t$:

$$\boxed{(A_a + x_t x_t^\top)^{-1} = A_a^{-1} - \frac{A_a^{-1} x_t x_t^\top A_a^{-1}}{1 + x_t^\top A_a^{-1} x_t}}$$
Cost breakdown per round:
  • Compute $A_a^{-1} x_t$: one matrix-vector multiply, $O(d^2)$
  • Compute $x_t^\top (A_a^{-1} x_t)$: one dot product, $O(d)$
  • Outer product update to $A_a^{-1}$: $O(d^2)$
  • Total: $O(d^2)$ per round vs $O(d^3)$ for full inversion
At $d=100$: $100^2 = 10{,}000$ vs $100^3 = 1{,}000{,}000$ operations — 100× speedup.
Proof of Sherman-Morrison

Let $B = A + uv^\top$. We claim $B^{-1} = A^{-1} - \frac{A^{-1}uv^\top A^{-1}}{1 + v^\top A^{-1}u}$. Verify by multiplying:

$$B \cdot B^{-1} = (A + uv^\top)\left(A^{-1} - \frac{A^{-1}uv^\top A^{-1}}{1+v^\top A^{-1}u}\right)$$ $$= I - \frac{uv^\top A^{-1}}{1+v^\top A^{-1}u} + \frac{uv^\top A^{-1}}{1+v^\top A^{-1}u} - \frac{u(v^\top A^{-1}u)(v^\top A^{-1})}{1+v^\top A^{-1}u}$$ $$= I + u v^\top A^{-1}\left(\frac{-1}{1+v^\top A^{-1}u} + \frac{1}{1+v^\top A^{-1}u} - \frac{v^\top A^{-1}u}{1+v^\top A^{-1}u}\right) = I \;\checkmark$$

Full algorithm state per arm $a$: Store $A_a^{-1}$ (updating via Sherman-Morrison) and $b_a$ (direct vector update). Compute $\hat\theta_a = A_a^{-1}b_a$ lazily at evaluation time.

The key question: how wrong could $\hat\theta_a$ be? The answer is an ellipsoidal confidence set in parameter space.

Self-Normalized Concentration (Abbasi-Yadkori et al., 2011)

Define the weighted norm $\|v\|_M = \sqrt{v^\top M v}$ for a positive definite matrix $M$. Then with probability $\geq 1-\delta$, for all $t \geq 0$ simultaneously:

$$\boxed{\|\hat\theta_a - \theta_a^*\|_{A_t} \leq \beta_t(\delta)}$$ where: $$\beta_t(\delta) = R\sqrt{d\ln\!\left(\frac{1+t/\lambda}{\delta}\right)} + \sqrt{\lambda}\,S$$

Here $R$ is the sub-Gaussian constant of the noise, $S = \|\theta_a^*\|_2$, and $t$ is the number of pulls of arm $a$.

What this means geometrically: The confidence set $$\mathcal{C}_t = \bigl\{\theta \in \mathbb{R}^d : \|\theta - \hat\theta_a\|_{A_t} \leq \beta_t(\delta)\bigr\}$$ is an ellipsoid in parameter space, centered at $\hat\theta_a$, with shape determined by $A_t$. It contains $\theta_a^*$ with probability $\geq 1-\delta$.

Shape of the ellipsoid: The set $\{v : v^\top A_t v \leq c^2\}$ is elongated in directions where $A_t$ has small eigenvalues — precisely the directions of $\mathbb{R}^d$ where we have seen few context vectors. In directions richly covered by data, $A_t$ is large, the ellipsoid is narrow, and we are confident.

From Parameter Uncertainty to Prediction Uncertainty

We care about prediction uncertainty: how uncertain is $x_t^\top\hat\theta_a$ as an estimate of $x_t^\top\theta_a^*$? By Cauchy-Schwarz applied to the weighted inner product:

$$|x_t^\top(\hat\theta_a - \theta_a^*)| = |\langle x_t, \hat\theta_a - \theta_a^*\rangle| \leq \|x_t\|_{A_t^{-1}} \cdot \|\hat\theta_a - \theta_a^*\|_{A_t}$$

where we used the identity $|\langle u,v\rangle| \leq \|u\|_{M^{-1}} \|v\|_M$ (dual norms). Combining with the confidence bound:

$$\boxed{|x_t^\top\hat\theta_a - x_t^\top\theta_a^*| \leq \beta_t(\delta)\cdot\sqrt{x_t^\top A_t^{-1} x_t}}$$

The term $\sqrt{x_t^\top A_t^{-1} x_t}$ is the half-width of the prediction confidence interval at context $x_t$. It is:

  • Large when $x_t$ points in a direction not covered by past contexts (unexplored territory)
  • Small when $x_t$ is similar to many past contexts (well-explored)

The UCB principle: always be optimistic — pull the arm whose upper confidence bound on its expected reward is highest.

LinUCB Score

For arm $a$ at round $t$ with context $x_t$:

$$\boxed{\text{UCB}(a,t) = \underbrace{x_t^\top\hat\theta_a}_{\text{estimated reward}} + \;\alpha\underbrace{\sqrt{x_t^\top A_a^{-1}x_t}}_{\text{uncertainty / exploration bonus}}}$$

Why does this achieve low regret? If we set $\alpha = \beta_t(\delta)$, then by the confidence bound from Tab 4:

$$x_t^\top\theta_a^* \leq x_t^\top\hat\theta_a + \beta_t(\delta)\sqrt{x_t^\top A_a^{-1}x_t} = \text{UCB}(a,t)$$

So UCB is an upper bound on the true reward. The algorithm therefore selects $A_t = \arg\max_a \text{UCB}(a,t)$.

Where Does $\alpha$ Come From?

Theoretically, $\alpha = \beta_t(\delta)$ from Abbasi-Yadkori et al. (2011):

$$\alpha_{\text{theory}} = R\sqrt{d\ln\!\left(\tfrac{1+t/\lambda}{\delta}\right)} + \sqrt{\lambda}\,S$$

In the original Li et al. (2010) paper, $\alpha$ is derived from Hoeffding's inequality for bounded rewards:

$$\alpha_{\text{Li}} = 1 + \sqrt{\tfrac{\ln(2/\delta)}{2}} \approx 1.41 \text{ at } \delta=0.05$$

In practice, $\theta_a^*$ is unknown so $S$ can't be computed. $\alpha$ is treated as a hyperparameter, tuned via cross-validation on logged data. Values in $[0.1, 1.0]$ typically work well.

Interpreting $\alpha$:
  • $\alpha \to 0$: pure exploitation — always pick the arm with highest estimated reward
  • $\alpha \to \infty$: pure exploration — pick the arm with highest uncertainty
  • Optimal $\alpha$ balances these, and scales as $\sqrt{d\log T}$ with dimension and time horizon

Regret Bound

With $\alpha = \beta_t(\delta)$, the cumulative regret satisfies with probability $\geq 1-\delta$:

$$\text{Regret}(T) \leq 2\beta_T(\delta)\sqrt{T \cdot d\ln(1+T/\lambda)}$$

This gives $\text{Regret}(T) = \tilde{O}(d\sqrt{T})$, where $\tilde{O}$ hides logarithmic factors. This matches the lower bound $\Omega(d\sqrt{T})$ — LinUCB is minimax optimal.

Complete LinUCB Algorithm (Disjoint)

Initialize: For each arm a = 1,...,K:
  A_a = λI ∈ ℝ^{d×d}         # Regularized covariance
  b_a = 0  ∈ ℝ^d              # Reward-weighted contexts

For t = 1, 2, ..., T:
  Observe context x_t ∈ ℝ^d

  For each arm a:
    θ̂_a     = A_a⁻¹ b_a               # Ridge regression estimate
    p_a(t)  = x_t^T θ̂_a               # Estimated reward
    cb_a(t) = α √(x_t^T A_a⁻¹ x_t)   # Confidence bound (exploration bonus)

  Pull arm A_t = argmax_a  [p_a(t) + cb_a(t)]
  Observe reward r_t

  # Update chosen arm only:
  A_{A_t} ← A_{A_t} + x_t x_t^T      # Rank-1 update (use Sherman-Morrison for A⁻¹)
  b_{A_t} ← b_{A_t} + r_t x_t

Disjoint LinUCB

Each arm $a$ has its own independent parameter vector $\theta_a \in \mathbb{R}^d$. Data for arm 1 says nothing about arm 2. The reward model is:

$$\mathbb{E}[r \mid x, a] = x^\top \theta_a^*$$

Best when: Arms are fundamentally different (different content types, different products). $K$ is small enough that each arm accumulates sufficient data. $d$ is moderate.

Hybrid LinUCB

Decomposes features into shared ($z \in \mathbb{R}^k$, e.g. user features) and arm-specific ($x \in \mathbb{R}^d$):

$$\mathbb{E}[r \mid x, z, a] = z^\top \beta^* + x^\top \theta_a^*$$

where $\beta^* \in \mathbb{R}^k$ is a global shared parameter estimated from all arms' data jointly. Maintain:

  • One global matrix $A_0 \in \mathbb{R}^{k\times k}$ and vector $b_0 \in \mathbb{R}^k$ for $\beta^*$
  • Per-arm matrices $A_a$ and vectors $B_a$ (cross-terms) for $\theta_a^*$

The UCB score becomes (more complex, involving cross-covariance):

$$\text{UCB}_{\text{hybrid}}(a,t) = z_t^\top\hat\beta + x_t^\top\hat\theta_a + \alpha\sqrt{s_{a,t}}$$

where $s_{a,t}$ accounts for uncertainty in both $\hat\beta$ and $\hat\theta_a$.

Best when: Many arms ($K$ large), per-arm data is sparse (cold-start), arms share structure (e.g. all articles have user-article interaction features).

Disjoint LinUCB
  • $K \times d \times d$ matrices stored
  • No cross-arm information sharing
  • Simpler update equations
  • Better when arms are truly independent
Hybrid LinUCB
  • $1$ global + $K$ arm-specific matrices
  • User features improve all arms
  • More complex update (cross-terms)
  • Better for large $K$, cold-start arms

Thompson Sampling for Contextual Bandits

Bayesian linear regression meets bandit exploration

Key insight: LinUCB uses a deterministic UCB score for exploration. Thompson Sampling (TS) achieves exploration implicitly — by sampling from the posterior over $\theta_a$ and acting greedily on the sample. Wide posteriors (high uncertainty) naturally produce varied samples, driving exploration. No $\alpha$ to tune.

We adopt the full Bayesian perspective on the same linear reward model. Place a Gaussian prior on each arm's parameter:

$$\theta_a \sim \mathcal{N}(0,\, \lambda^{-1}I)$$

This encodes the belief that $\theta_a^*$ has entries of magnitude $\sim 1/\sqrt\lambda$ before any data.

With a Gaussian likelihood (conditional on $\theta_a$):

$$r \mid x, \theta_a \sim \mathcal{N}(x^\top\theta_a,\, \sigma^2)$$

The prior and likelihood are conjugate — the posterior is also Gaussian. This enables closed-form Bayesian updates with no Monte Carlo needed.

Conjugacy: Gaussian prior + Gaussian likelihood = Gaussian posterior. This is the key computational advantage of the linear bandit model. The posterior mean is exactly the ridge regression estimate, and the posterior covariance is exactly $A_a^{-1}$.

After observing pulls $\{(x_s, r_s)\}_{s \in \mathcal{T}_a}$ for arm $a$, the posterior is:

$$\theta_a \mid \mathcal{D}_t \sim \mathcal{N}(\mu_a,\, \Sigma_a)$$

Computing $\Sigma_a^{-1}$ by combining prior precision and likelihood precision:

$$\Sigma_a^{-1} = \underbrace{\lambda I}_{\text{prior precision}} + \underbrace{\frac{1}{\sigma^2}\sum_{s \in \mathcal{T}_a} x_s x_s^\top}_{\text{likelihood precision}} = \frac{1}{\sigma^2}\underbrace{\Bigl(\lambda\sigma^2 I + \sum_s x_s x_s^\top\Bigr)}_{A_a \text{ (if } \lambda_{\text{eff}} = \lambda\sigma^2\text{)}}$$

Setting $\sigma^2 = 1$ (absorbing into $\lambda$), we get:

$$\boxed{\Sigma_a = A_a^{-1}, \qquad \mu_a = A_a^{-1}b_a = \hat\theta_a}$$
Remarkable fact: The TS posterior mean is identical to the LinUCB point estimate $\hat\theta_a$, and the posterior covariance is $A_a^{-1}$ — the same matrix used in the LinUCB exploration bonus. Both algorithms share the same sufficient statistics $(A_a, b_a)$. They differ only in how they use them.

Incremental update: On new observation $(x_t, r_t)$ for arm $a$:

$$A_a \leftarrow A_a + x_tx_t^\top, \quad b_a \leftarrow b_a + r_tx_t$$ $$\Sigma_a \leftarrow A_a^{-1} \text{ (via Sherman-Morrison)}, \quad \mu_a \leftarrow A_a^{-1}b_a$$

The posterior narrows (covariance shrinks) in directions where many contexts have been observed, reflecting decreased uncertainty.

The Thompson Sampling Algorithm

At each round $t$ with context $x_t$:

Sample
$\tilde\theta_a \sim \mathcal{N}(\mu_a, \nu^2\Sigma_a)$
for each arm $a$
→
Select arm
$A_t = \arg\max_a x_t^\top\tilde\theta_a$
→
Observe $r_t$,
update $A_{A_t}$, $b_{A_t}$

The scaling $\nu^2$ plays the role of $\alpha$ in LinUCB. Agrawal & Goyal (2013) show $\nu = R\sqrt{9d\ln(T/\delta)}$ achieves $\tilde{O}(d\sqrt{T})$ regret, but $\nu = 1$ often works well in practice.

Intuition: Why Does Sampling Achieve Exploration?

Consider arm $a$ that has been pulled infrequently. Its posterior $\mathcal{N}(\mu_a, \Sigma_a)$ has high variance — samples $\tilde\theta_a$ will be spread far from $\mu_a$, occasionally yielding an optimistic sample where $x_t^\top\tilde\theta_a$ is large, causing arm $a$ to be selected. This is exploration: the algorithm tries arms proportionally to how often they could be the best arm under the posterior.

As data accumulates, $\Sigma_a \to 0$, samples concentrate near $\mu_a \approx \theta_a^*$, and the algorithm increasingly exploits.

Computational Note

Sampling from $\mathcal{N}(\mu_a, \nu^2 A_a^{-1})$ requires the Cholesky decomposition $L L^\top = A_a^{-1}$:

$$\tilde\theta_a = \mu_a + \nu\, L\, z, \quad z \sim \mathcal{N}(0, I)$$

This costs $O(d^3)$ per arm per round in general. For large $d$, approximations are used: last-layer only (treat lower layers as fixed feature extractor), diagonal $\Sigma_a$ (ignores correlations), or ensemble/bootstrap methods that avoid explicit covariance.

Why TS often wins empirically: LinUCB's exploration bonus $\alpha\sqrt{x^\top A^{-1}x}$ scales identically across all arms with the same uncertainty — it doesn't differentiate based on what's plausible for the reward. TS naturally selects arms in proportion to the probability they are optimal, which is more efficient. It also handles multi-modality in the posterior better.

Neural Extensions

Both algorithms generalize naturally to nonlinear reward models:

  • NeuralUCB: Replace $x$ with the gradient of a neural net $\nabla_\theta f(x;\theta) \in \mathbb{R}^p$. Maintain $Z_t = \lambda I + \sum_s g_s g_s^\top$ in gradient space. UCB = $f(x;\theta) + \beta\sqrt{g(x)^\top Z_t^{-1}g(x)}$. Justified via neural tangent kernel theory.
  • NeuralTS: Sample from posterior in NTK-induced RKHS. Same $\tilde{O}(\sqrt{T})$ regret. Outperforms NeuralUCB empirically.
  • Bootstrapped bandits: Train $M$ networks on bootstrap resamples; use variance across heads as uncertainty. Compatible with any architecture.

Applications & Tooling

Yahoo! News — The LinUCB Deployment

Li et al. (2010) deployed LinUCB on the Yahoo! Front Page "Today Module" — selecting which article to show from ~20 candidates to maximize click-through rate (CTR). Each page load: observe a 6-dim user feature vector, select an article (arm). The article has a 6-dim feature vector. Hybrid LinUCB forms a 36-dim user-article interaction feature.

Result: +12.5% CTR improvement over a random policy in live A/B tests on ~4.7M events. The paper also introduced the importance-weighted offline evaluation methodology that is now standard.

Offline Evaluation: You Only See What You Chose

A core practical challenge: you have logged data from an old policy $\pi_b$ and want to evaluate a new policy $\pi_e$ offline. The problem: $\pi_e$ might choose arm 3, but your logs only have reward observations for whichever arm $\pi_b$ chose. Counterfactual rewards are missing.

MethodFormulaBiasVarianceRequires
Direct Method (DM) $\frac{1}{n}\sum_t \hat r(x_t, \pi_e(x_t))$ High if $\hat r$ wrongLowReward model $\hat r$
IPS $\frac{1}{n}\sum_t r_t \frac{\mathbf{1}[a_t=\pi_e(x_t)]}{\pi_b(a_t|x_t)}$ Zero (unbiased)High (clips at $M$)Logging propensities $\pi_b$
Doubly Robust (DR) DM + IPS $\cdot$ (residual) Low (consistent if either model correct)MediumBoth $\hat r$ and $\pi_b$

The Doubly Robust estimator (Dudík et al., 2011) combines both:

$$\hat V_{DR}(\pi_e) = \hat V_{DM}(\pi_e) + \frac{1}{n}\sum_t \frac{\mathbf{1}[a_t=\pi_e(x_t)]}{\pi_b(a_t|x_t)}\bigl(r_t - \hat r(x_t, a_t)\bigr)$$

DR is consistent if either the reward model $\hat r$ or the logging propensity $\pi_b$ is correctly specified — it's doubly robust to misspecification of one of the two.

Bandits vs. Full RL — Decision Guide

Use contextual bandits when:
  • Reward is immediate and directly observable
  • Your action does not meaningfully change future contexts
  • You need sub-millisecond inference ($K$ small, $d$ moderate)
  • Rich context, instant feedback: ads, recommendations, pricing
Use full RL when:
  • Actions change state (chatbots, tutoring, robotics)
  • Delayed or sparse rewards (drug trials, games)
  • Long-horizon planning required
  • Session-level optimization (total watch time, not per-click CTR)
