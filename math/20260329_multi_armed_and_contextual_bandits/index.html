Multi-Armed & Contextual Bandits

An interactive explainer — from Thompson (1933) to neural bandits (2024)

The core problem: You face $K$ options ("arms"), each with an unknown reward distribution. You have $T$ total trials. How do you maximize cumulative reward — without committing too early to a seemingly good option, yet without wasting trials on obviously bad ones?

The fundamental tension — Explore vs. Exploit:
  • Exploit — pull the arm with the highest estimated reward so far
  • Explore — try other arms; one might actually be better

Too much exploitation → stuck on a suboptimal arm forever. Too much exploration → burning trials on arms you already know are bad.

The key metric is regret — the cumulative loss versus an oracle that always pulls the best arm:

$$\text{Regret}(T) = T\mu^* - \sum_{t=1}^{T} r_t$$

where $\mu^*$ is the best arm's true mean reward. Good algorithms achieve logarithmic regret — the per-step loss shrinks as learning accumulates.

⚠️ Bandits ≠ full Reinforcement Learning. Each bandit round is stateless — your action does not change the state of the world. This simplification makes bandits tractable for real-time production systems.

🎲 The Bandit Problem

Formal setup, regret decomposition, and theoretical limits

Formal Setup

A stochastic $K$-armed bandit is defined by:

  • $K$ arms, indexed $i = 1, \ldots, K$
  • Each arm $i$ has an unknown reward distribution $\nu_i$ with mean $\mu_i$
  • At each round $t$: agent selects $A_t \in \{1,\ldots,K\}$, receives $R_t \sim \nu_{A_t}$
  • Goal: maximize $\sum_{t=1}^T R_t$ over $T$ rounds
Regret decomposition: $$\text{Regret}(T) = \sum_{t=1}^T \!\left(\mu^* - \mu_{A_t}\right) = \sum_{i=1}^K \Delta_i \cdot \mathbb{E}[N_i(T)]$$

where $\Delta_i = \mu^* - \mu_i$ is the gap of arm $i$ and $N_i(T)$ is how often arm $i$ was pulled. Each suboptimal pull costs exactly $\Delta_i$.

An arm with a large gap $\Delta_i$ is obviously bad — we learn to avoid it quickly. An arm with a small gap $\Delta_i \approx 0$ is nearly optimal and worth exploring. This motivates concentrating exploration near the top arms.

Why Uncertainty Matters

Consider two arms, each pulled 5 times:

After 100 more pulls of each:

ArmPullsEstimate $\hat\mu$Should we explore?
A1050.58Low — we know it well
B1050.22Very low — confidently bad
The key insight: exploration need should decrease as pulls accumulate. ε-greedy ignores this (always explores with prob. ε). UCB1 and Thompson Sampling tie exploration directly to remaining uncertainty.
📐 Theoretical Lower Bound — Lai & Robbins (1985)

Lai and Robbins proved that any consistent algorithm must satisfy:

$$\liminf_{T\to\infty} \frac{\mathbb{E}[\text{Regret}(T)]}{\ln T} \;\geq\; \sum_{i:\,\Delta_i > 0} \frac{\Delta_i}{\text{KL}(\nu_i \,\|\, \nu^*)}$$

where $\text{KL}(\nu_i \| \nu^*)$ is the KL-divergence between arm $i$'s distribution and the optimal arm's. This establishes that logarithmic regret is the best achievable. Algorithms that match this bound (UCB1, Thompson Sampling) are called asymptotically optimal. ε-greedy achieves only $O(\sqrt{T \ln K})$.

📝 Bernoulli Bandits — The Setup Used in This Explainer

All simulations here use Bernoulli bandits: each arm $i$ pays reward $R = 1$ with probability $p_i$ and $R = 0$ with probability $1 - p_i$. The unknown parameters are the probabilities $p_1, \ldots, p_K \in [0,1]$.

This is the natural model for click-through rates, conversion rates, and engagement signals — the dominant reward type in production recommendation systems. Thompson Sampling's Beta prior is the conjugate prior for Bernoulli likelihoods, which is why it fits so naturally here.

⚙️ Core Algorithms

Three fundamental strategies for the explore-exploit tradeoff

Idea

With probability $\varepsilon$, explore randomly. Otherwise, exploit the current best estimate:

$$A_t = \begin{cases} \text{uniform random arm} & \text{with prob. } \varepsilon \\ \arg\max_i \hat{\mu}_i & \text{with prob. } 1-\varepsilon \end{cases}$$

After observing reward $r_t$ for chosen arm $a$, update the running mean:

$$\hat{\mu}_a \;\leftarrow\; \hat{\mu}_a + \frac{1}{N_a}\bigl(r_t - \hat{\mu}_a\bigr)$$ 
eps = 0.1
counts = [0]*K;  mu_hat = [0.0]*K

for t in range(T):
    if random() < eps:
        arm = randint(0, K)      # explore
    else:
        arm = argmax(mu_hat)     # exploit

    reward = env.pull(arm)
    counts[arm] += 1
    mu_hat[arm] += (reward - mu_hat[arm]) / counts[arm]

Properties

✅ Strengths: Trivially simple to implement and explain. Works in non-stationary environments (just use decaying or sliding-window ε). No assumptions about reward structure.
⚠️ Weakness: Explores uniformly at random — pulls arms you already know are bad with the same probability as unknown arms. Regret is $O(\sqrt{T})$, not optimal $O(\log T)$.

Sensitivity to ε

εBehavior
ε = 0Pure greedy — commits to first apparent winner, never corrects
ε = 0.1Standard default — 10% random, 90% exploit
ε = 0.5Half random — very slow convergence
ε → 0 (annealed)Approaches optimal if schedule is $\varepsilon_t = c/(t+1)$

Idea: Optimism Under Uncertainty

Add an exploration bonus to each arm's estimate that is large when the arm is under-explored, and shrinks as pulls accumulate. Always select the arm with the highest upper confidence bound:

$$A_t = \arg\max_i \;\underbrace{\hat{\mu}_i}_{\text{exploit}} + \underbrace{\sqrt{\dfrac{2\ln t}{N_i(t-1)}}}_{\text{explore bonus}}$$

Pull each arm once at initialization, then apply UCB from round $K+1$ onward.

# Initialize: pull each arm once
for i in range(K):
    counts[i] = 1
    mu_hat[i] = env.pull(i)

# Main loop
for t in range(K, T):
    ucb_scores = [mu_hat[i] + sqrt(2*log(t)/counts[i])
                  for i in range(K)]
    arm = argmax(ucb_scores)
    reward = env.pull(arm)
    counts[arm] += 1
    mu_hat[arm] += (reward - mu_hat[arm]) / counts[arm]

Properties

✅ Strengths: Deterministic — fully reproducible, auditable, cannot be gamed by adversarial environments. Achieves optimal $O(\log T)$ regret. No tuning required.
⚠️ Weakness: The confidence bonus is somewhat conservative — UCB1 over-explores arms that are close in value. Empirically outpaced by Thompson Sampling in most benchmarks.

Why does the bonus work?

By Hoeffding's inequality, the true mean $\mu_i$ lies below $\hat{\mu}_i + \sqrt{2\ln t / N_i}$ with probability at least $1 - 1/t^4$. Choosing the highest UCB is equivalent to asking: "Which arm could plausibly be optimal right now?" — and committing to it. Uncertainty naturally collapses as $N_i \to \infty$.

Idea: Sample from Posterior Beliefs

Maintain a probability distribution over each arm's unknown reward probability. At each round, sample one value from each distribution and pull the arm with the highest sample:

$$\theta_i \sim \text{Beta}(\alpha_i,\, \beta_i), \qquad A_t = \arg\max_i \theta_i$$

Start with $\alpha_i = \beta_i = 1$ (uniform prior — total ignorance). Update after each pull:

$$\text{reward}=1:\quad \alpha_{A_t} \mathrel{+}= 1 \qquad \text{reward}=0:\quad \beta_{A_t} \mathrel{+}= 1$$ 
alpha = [1]*K;  beta = [1]*K

for t in range(T):
    # Sample one value per arm from its current belief
    theta = [sample_beta(alpha[i], beta[i]) for i in range(K)]
    arm = argmax(theta)

    reward = env.pull(arm)

    if reward == 1: alpha[arm] += 1
    else:           beta[arm]  += 1

Properties

✅ Strengths: Matches optimal $O(\log T)$ regret. Empirically best among the three algorithms. Handles non-stationary environments naturally. The Beta distribution is the statistically correct model for Bernoulli rewards.
⚠️ Weakness: Stochastic — different runs yield different results. Requires a tractable posterior; for complex reward functions, approximations (bootstrapping, Laplace) are needed.

What Beta($\alpha$, $\beta$) represents

DistributionMeaningMean
Beta(1, 1)Uniform — total ignorance0.50
Beta(2, 8)10 obs, 2 wins — likely bad0.20
Beta(8, 2)10 obs, 8 wins — likely good0.80

🧠 Contextual Bandits

Personalizing decisions with side information

The Key Extension

Standard MAB finds one global best arm. Contextual bandits ask: "What is the best arm for this specific situation?"

Contextual bandit loop (each round $t$):
  1. Observe context $x_t \in \mathbb{R}^d$ (a feature vector)
  2. Choose arm $A_t = \pi(x_t)$ using learned policy $\pi$
  3. Observe reward $r_t \sim p(\cdot \mid x_t, A_t)$
  4. Update policy using the triple $(x_t, A_t, r_t)$

The reward now depends on both who is asking and what you show them. The algorithm learns a mapping from context to optimal arm, rather than a single universal winner.

DomainContext Features $x_t$
E-commercePurchase history, device type, time of day
HealthcarePatient vitals, medication history, symptoms

Behavioral healthEngagement signals, session history, recent mood indicators

LinUCB Algorithm

Assumes reward is a linear function of context: $\mathbb{E}[r \mid x, a] = x^\top \theta_a$

Each arm $a$ maintains covariance matrix $A_a \in \mathbb{R}^{d\times d}$ and accumulator $b_a \in \mathbb{R}^d$:

$$\hat{\theta}_a = A_a^{-1} b_a$$ $$\text{Score}(a) = \underbrace{x_t^\top \hat{\theta}_a}_{\text{estimated reward}} + \;\alpha \underbrace{\sqrt{x_t^\top A_a^{-1} x_t}}_{\text{uncertainty bonus}}$$

Choose $A_t = \arg\max_a \text{Score}(a)$, then update:

$$A_a \leftarrow A_a + x_t x_t^\top, \qquad b_a \leftarrow b_a + r_t\, x_t$$
The uncertainty term $\sqrt{x_t^\top A_a^{-1} x_t}$ is large when $x_t$ is far from previously seen contexts — the arm earns an exploration bonus whenever it encounters a novel situation, exactly where we should be least confident.
🧮 Contextual Thompson Sampling & Neural Variants

Contextual TS: Maintain posterior $\theta_a \sim \mathcal{N}(\hat\theta_a, A_a^{-1})$ per arm. At each round, sample $\tilde\theta_a \sim \mathcal{N}(\hat\theta_a, A_a^{-1})$ and pick $a = \arg\max_a\, x_t^\top \tilde\theta_a$.

Neural contextual bandits (NeuralUCB/NeuralTS): Replace the linear reward model with a neural network. Uncertainty comes from the final layer's covariance. Enables learning non-linear context–reward relationships at the cost of more computation and data.

LLM-enhanced bandits (2024+): Use a language model to generate or enrich the context vector $x_t$ from unstructured data (chat logs, search history). Dramatically expands expressiveness of the context representation.

MAB vs. Contextual Bandit — Side by Side

Multi-Armed Bandit

Finds one globally best arm across all users and situations.

Policy: $A_t = f(\text{history of pulls})$

Analogy: Classic A/B test — one winning variant for all users

When to use: Homogeneous users; no meaningful feature signal

Contextual Bandit

Finds the best arm per individual given their observed context.

Policy: $A_t = f(x_t,\, \text{history})$

Analogy: Personalized recommendation — different variant per user

When to use: Heterogeneous users; rich features available

💡 Critical practitioner insight: Feature engineering typically outweighs algorithm choice. A well-designed context vector with LinUCB often beats a poorly-designed one with NeuralUCB. Start simple, add complexity only when improvement is measurable.
