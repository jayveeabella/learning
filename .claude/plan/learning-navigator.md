# Implementation Plan: Learning Navigator Web Server

## Overview

A lightweight Python/Flask web server that lives at the root of the `learning` repo.
It auto-discovers every `index.html` inside dated topic folders, parses their metadata,
and serves a clean homepage for navigation — designed to run on a Raspberry Pi.

---

## Directory Convention (current + future)

```
learning/
├── server.py                          ← new: web server
├── pyproject.toml                     ← new: root-level deps (Flask)
├── learning.service                   ← new: systemd unit (Pi autostart reference)
├── math/
│   ├── 20260327_random_fourier_features/index.html
│   └── 20260328_johnson_lindenstrauss_lemma/index.html
├── physics/                           ← future field
│   └── 20260401_fourier_transform/index.html
└── ...
```

Folder naming convention: `YYYYMMDD_snake_case_topic`
Top-level directory = field (math, physics, cs, …)

---

## Technical Solution

**Stack**: Python 3.11+ · Flask · uv (already used in the project)

**Why Flask over bare `http.server`**: Need one dynamic route for the homepage;
all other routes are static file serving. Flask adds ~2 MB and zero config overhead.

## Implementation Steps

### Step 1 — `pyproject.toml` (repo root)

```toml
[project]
name = "learning-navigator"
version = "0.1.0"
requires-python = ">=3.11"
dependencies = [
    "flask>=3.0",
]
```

### Step 2 — `server.py`

**`discover_pages()` logic:**
```python
ROOT = Path(__file__).parent
pattern = re.compile(r'^(\d{8})_(.+)$')

for html in ROOT.glob('*/*/index.html'):
    field = html.parts[-3]          # "math"
    folder = html.parts[-2]         # "20260328_johnson_lindenstrauss_lemma"
    m = pattern.match(folder)
    date  = datetime.strptime(m[1], '%Y%m%d')  if m else None
    title = m[2].replace('_', ' ').title()      if m else folder
    url   = f'/{field}/{folder}/index.html'
    → append to pages[field]

sort each field by date descending
```

**Routes:**
- `GET /` → render homepage (inline Jinja2 template)
- `GET /<path:filepath>` → `send_from_directory(ROOT, filepath)`

**Homepage template** (inline, dark theme matching existing pages):
- `#1a1a2e` background, `#c8d0e7` text — same CSS vars used in the HTML files
- Field sections as collapsible groups
- Each entry: date chip + title + "Open" link
- No external dependencies (pure CSS, no JS framework)
- Responsive grid for future fields

**Server config:**
```python
if __name__ == '__main__':
    port = int(os.environ.get('PORT', 8080))
    app.run(host='0.0.0.0', port=port, debug=False)
```

### Step 3 — `learning.service` (systemd unit for Raspberry Pi)

```ini
[Unit]
Description=Learning Navigator Web Server
After=network.target

[Service]
Type=simple
User=pi
WorkingDirectory=/home/pi/repos/learning
ExecStart=/home/pi/.local/bin/uv run python server.py
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

## Raspberry Pi Deployment Steps

1. **Clone the repo on the Pi**
   ```bash
   git clone <repo-url> ~/repos/learning
   cd ~/repos/learning
   ```

2. **Install uv** (if not already)
   ```bash
   curl -LsSf https://astral.sh/uv/install.sh | sh
   ```

3. **Run the server**
   ```bash
   uv run python server.py
   # → listening on 0.0.0.0:8080
   ```

4. **Autostart with systemd** (optional but recommended)
   ```bash
   sudo cp learning.service /etc/systemd/system/
   sudo systemctl daemon-reload
   sudo systemctl enable learning
   sudo systemctl start learning
   ```

5. **Access via Raspberry Pi Connect**
   - Pi Connect tunnels to the Pi; once connected, browse to `http://localhost:8080`
   - Or use the Pi's local IP on your home network: `http://raspberrypi.local:8080`

## Risks and Mitigation

| Risk | Mitigation |
|------|------------|
| New folders don't match `YYYYMMDD_` pattern | Graceful fallback: use folder name as title, no date shown |
| Future fields other than `math/` | Discovery uses `*/*/index.html` glob — works for any top-level field folder |
| Port 8080 already in use on Pi | `PORT=9000 uv run python server.py` env var override |
| Static assets (JS/CSS) referenced by index.html | `send_from_directory` serves entire repo tree, so all relative paths resolve |
